configuration

configuration files (dir: SpectrHA/config):

S_settings.m
- main interface colors
- markers and colors
- fonts
- default xlim (default spectrum xlim, to use with the button [default xlim], see gui)
- default datenum vars (used in prediction)
- default ellipse color (to use when not specified)
- default NetCDF file extensions (to use in uigetfile when loading grid and stations file)
- default log filename (otherwise call spectrha(logname))
- default time index when release s_levels (itime)

S_theme.m
colors, markers and fonts of other theme (change theme: [SpectrHA]-->[preferences]-->[theme]-->...)

S_obj_positions.m
positions of objects in the gui

S_init_global.m
initialization of global variables

set_data.m
way of get variables, change this if you are not using ROMS NetCDF output, but some other kind of data

S_get_ncvar.m
way of load NetCDF variables, change this if you are not using ROMS output

S_config_LSF.m
LSF configuration:
- start and end time index, to use only part of data.
This can be usefull if first days are affected by some kind of ramp.
- names or frequencies (periods) of tidal constituents to use in LSF
- colors to use in ellipses

S_config_T_TIDE.m
T_TIDE configuration:
- start and end time index
- colors to use in ellipses
- all t_tide options
- use or not stations and data file latitude
- use or not stations and data file start_time
- t_predic synthesis
- t_predic latitude
